Apps Monitor

API — Register an App

Copy and give to AI or run in terminal. One POST auto-creates: port, Caddy proxy, LaunchAgent, health check, and log path.

Minimal (just id + path)
curl -X POST http://localhost:9876/api/apps \ -H "Content-Type: application/json" \ -d '{"id":"APP_NAME","localPath":"/Users/USERNAME/Sites/APP_NAME"}'
Full (override port, start command, repo)
curl -X POST http://localhost:9876/api/apps \ -H "Content-Type: application/json" \ -d '{ "id": "APP_NAME", "name": "APP_NAME", "localUrl": "http://localhost:PORT", "localPath": "/Users/USERNAME/Sites/APP_NAME", "repo": "https://github.com/USERNAME/APP_NAME", "startCommand": "npm run dev" }'
AI Instruction (copy this to any AI)
Register this app with my local apps-monitor by making a POST request: POST http://localhost:9876/api/apps Content-Type: application/json Required fields: - "id": app name (lowercase, kebab-case) - "localPath": absolute path to the project root Optional fields (auto-generated if omitted): - "localUrl": defaults to next available port (http://localhost:PORT) - "healthUrl": defaults to localUrl - "repo": GitHub URL - "startCommand": defaults to "npm run dev" (also supports "npm start", "bun dev", etc.) - "logPath": defaults to /tmp/{id}.log The monitor will automatically: 1. Assign a dedicated port if not provided 2. Create a Caddy reverse proxy at http://{id}.localhost 3. Create a macOS LaunchAgent plist 4. Begin health-checking every 10 seconds Example: curl -X POST http://localhost:9876/api/apps \ -H "Content-Type: application/json" \ -d '{"id":"my-app","localPath":"/Users/me/Sites/my-app"}'
